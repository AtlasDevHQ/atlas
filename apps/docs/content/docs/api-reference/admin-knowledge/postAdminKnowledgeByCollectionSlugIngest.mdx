          Upload a `.tar`, `.tar.gz`, or `.zip` OKF bundle into a collection.
          Documents are parsed leniently (plain markdown works — missing OKF
          frontmatter is stamped) and upserted by path at `status='draft'`.
          Unparseable / oversized / unsafe-path files are rejected with per-file
          errors. Pass `?publish=true` to atomically run the workspace publish
          in the same action — this promotes ALL pending drafts in the workspace
          (matching the atomic publish endpoint), not only the just-ingested
          documents.
