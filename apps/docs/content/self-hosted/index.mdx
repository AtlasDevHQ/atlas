Every page in this section is written for operators running Atlas on their own
infrastructure. Atlas is AGPL-licensed and free to self-host — you control the
environment, data residency, and LLM provider.

## Start here

- **[Quick Start](/self-hosted/getting-started/quick-start)** — get Atlas running locally in under 5 minutes with the demo dataset.
- **[Deploy](/self-hosted/deployment/deploy)** — one-click deploy to Railway, Vercel, or Docker, or scaffold your own project.
- **[Authentication](/self-hosted/deployment/authentication)** — configure API-key, managed, or bring-your-own-token (JWKS) auth.

## Operate

- **[Self-Hosted Models](/self-hosted/guides/self-hosted-models)** — run Atlas against Ollama, vLLM, or any OpenAI-compatible endpoint.
- **[Cache Configuration](/self-hosted/deployment/cache-configuration)** — tune query-result caching or plug in an external Redis backend.
- **[Bring Your Own Frontend](/self-hosted/frameworks/overview)** — embed the headless Atlas API in React, Vue, Svelte, or TanStack.
- **[Contributing](/self-hosted/contributing/ci)** — CI gates and the eval harness for working on Atlas itself.

Shared concept pages — the semantic layer, plugins, SDK, and integrations —
will be single-sourced into this section from `content/shared/` in later
migration slices, so a self-hoster won't get bounced into a SaaS-framed page
mid-task.
