Atlas exposes a JSON + streaming API over HTTP. All endpoints are served by the Hono API server (standalone on `:3001`, or proxied through Next.js on `:3000`).

## Base URL

| Setup | Base URL |
|-------|----------|
| Same-origin (Next.js rewrites) | `/api` |
| Cross-origin (standalone API) | `http://localhost:3001/api` |
| Production | `https://api.useatlas.dev/api` |

## Authentication

Authentication depends on the configured `ATLAS_AUTH_MODE`:

| Mode | Header |
|------|--------|
| `none` | No auth required |
| `api-key` | `Authorization: Bearer <ATLAS_API_KEY>` |
| `managed` | Session cookie (Better Auth) |
| `byot` | `Authorization: Bearer <JWT>` |

## Endpoints

### Core

- [**Chat**](/api-reference/chat/chatStream) — Stream agent responses via `POST /api/v1/chat`
- [**Query**](/api-reference/query/query) — Synchronous JSON query via `POST /api/v1/query`
- [**Conversations**](/api-reference/conversations/listConversations) — List, get, and delete conversations
- [**Health**](/api-reference/health/healthCheck) — Health check and readiness probe

### Features

- [**Actions**](/api-reference/actions/listActions) — Approval-gated write operations
- [**Scheduled Tasks**](/api-reference/scheduled-tasks/listScheduledTasks) — Recurring query schedules and delivery
- [**Auth**](/api-reference/auth/getSession) — Better Auth endpoints (signup, signin, session)
- [**Slack**](/api-reference/slack/slackInstall) — Slash commands, events, and OAuth
- [**Admin**](/api-reference/admin/adminOverview) — Admin console API (users, settings, connections)
