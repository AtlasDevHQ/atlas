Runs a single refresh cycle synchronously and returns the outcome counts. Audited via the standard `model_config.catalog_refresh_cycle` action with actor `system:byot-catalog-refresh`. Returns 500 with `requestId` if the cycle failed before producing per-row counts (e.g. the stale-row query itself threw).
