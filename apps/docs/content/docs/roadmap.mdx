Atlas uses public semver starting at `0.0.x`. For real-time tracking, see the [GitHub Project Board](https://github.com/orgs/AtlasDevHQ/projects/2).

## Shipped

### Public Launch

Initial open-source release with admin user management, one-click Vercel deploy (Neon + AI Gateway), `@useatlas` packages on npm, and CI automation for template drift checks and starter repo sync.

### Adapter Plugin Refactor

Datasource adapters extracted from core into standalone plugins. The Plugin SDK gained `parserDialect` and `forbiddenPatterns` hooks, SQL validation and the connection registry became plugin-aware, and official plugins shipped for ClickHouse, Snowflake, DuckDB, and Salesforce.

### Python Data Science Sandbox

A sandboxed `executePython` tool for data analysis — runs Python with import guards, renders charts inline in the chat UI, and works across all sandbox backends (nsjail, sidecar, Vercel).

### 0.1.0 — Documentation & Developer Experience

Docs site launched at [docs.useatlas.dev](https://docs.useatlas.dev) with reference pages, integration guides, interactive API reference from OpenAPI, OG images, sitemap, and llms.txt. CLI gained `atlas doctor`, `atlas validate`, shell completions, and better first-run errors. Test coverage expanded across web UI, SDK integration, and E2E suites. Added CHANGELOG, CONTRIBUTING guide, issue/PR templates, and unified brand colors.

### 0.2.0 — Plugin Ecosystem

Official plugins published to npm under `@useatlas/*` (ClickHouse, Snowflake, DuckDB, Salesforce, Slack, MCP, and more). Plugin SDK gained testing utilities, a scaffold command (`bun create @useatlas/plugin`), multi-type plugin support, and schema migrations. Docs site added per-plugin reference pages, a filterable plugin directory, a composition guide, and a plugin cookbook.

### 0.3.0 — Admin Console & Operations

Self-service management and observability for the admin console. Connection CRUD with encrypted URLs at rest, action approval with bulk operations, user invite flow, plugin and settings configuration. Query analytics dashboard, health dashboard with per-component checks, token usage tracking per-user and per-conversation, and OpenTelemetry traces. Scheduled tasks gained create/edit forms, run history with filters, and delivery channel configuration.

### 0.4.0 — Chat Experience

Dark/light mode toggle with persistence and admin-configurable brand color, suggested follow-up questions after each answer, Excel export alongside CSV, mobile-responsive chat UI, saved queries with starred filters, visual schema explorer for non-admin users, and expanded chart types (area, stacked bar, scatter).

### 0.5.0 — Launch

Embeddable widget infrastructure: `@useatlas/react` component with headless hooks API, `<script>` tag loader with programmatic JS API, and dedicated widget host route with configurable branding. SDK streaming via `streamQuery()` with abort support. BigQuery datasource plugin with cost-gated query approval. Conversation sharing with public links, OG meta tags, iframe embed mode, expiry controls, and org-scoped permissions. Launch prep including README overhaul with animated terminal recording, landing page refresh with interactive widget demo, and comparison pages. Extracted `@useatlas/types` shared package, hardened error handling across the stack, and isolated CLI test runner.

## 0.5.1 — Agent-Friendly Docs

The current milestone. Systematic docs overhaul applying best practices for both human and LLM consumption.

- **Tutorials** — Verification checkpoints after every setup step, prerequisites in expandable blocks, per-stage troubleshooting
- **How-to guides** — New guides for widget customization, multi-datasource routing, schema evolution, rate limiting. Corner cases moved out of tutorials into dedicated pages
- **Reference pages** — More code examples with inline comments, curl/fetch request examples, pagination and error handling patterns
- **Explanation pages** — SQL validation pipeline walkthrough, semantic layer concepts glossary, comparison tables (Widget vs SDK vs API, auth modes)
- **Agent optimization** — llms.txt audit, stable headings, no media-only information

---

## 0.5.2 — Onboarding & CLI Polish

Making the first 60 seconds perfect — every error a new user could hit gets a clear, actionable message.

- **Onboarding** — `atlas init` progress spinner with ETA, failure threshold for table profiling, first-run `.env` detection
- **Error handling** — Pattern-matched errors replacing generic fallbacks (connection refused, provider unavailable, query timeout), request IDs on internal errors, transient vs permanent error hints
- **CLI** — Proper `--help` for all subcommands, better config validation messages

---

## 0.5.3 — UI & Accessibility Polish

No new features — making everything feel right across the existing surface area.

- **Accessibility** — ARIA labels on chat input and message bubbles, skip links, keyboard navigation audit, Lighthouse/axe compliance
- **UI** — Loading state audit, meaningful empty states, error boundaries, mobile polish pass, chart responsiveness
- **Widget** — TypeScript type exports for embedders, CSS customization docs, graceful error states

---

## 0.5.4 — SDK & Integration Polish

Making every integration surface production-ready.

- **SDK** — `listTables()` convenience method, error code catalog with retry guidance, streaming timeout documentation
- **Testing** — End-to-end widget embed test, SDK streaming test, MCP server smoke test
- **Docs cross-cuts** — Troubleshooting section on every guide, self-contained how-to pages, cross-link audit

---

## 0.6.0 — Governance & Integrations

Access control, audit, and third-party integrations for production teams.

- **Access control** — Advanced RLS policies (multi-column, array claims), session management, OIDC/OAuth provider docs
- **Audit** — Audit log export and search, data classification tags on query trails
- **Integrations** — Microsoft Teams plugin, generic webhook plugin for Zapier/Make/n8n, email digest subscriptions
- **Data protection** — PII detection and column masking via plugin hooks

---

## 0.7.0 — Performance & Scale

Handle larger semantic layers, more concurrent users, and bigger result sets.

- **Caching** — Query result caching with configurable TTL, semantic layer indexing
- **Infrastructure** — Connection pool improvements, streaming Python output, tenant-scoped semantic layers

---

## 0.8.0 — Intelligence & Learning

An agent that gets smarter over time.

- **Knowledge** — Curated prompt library per industry, query suggestion engine from usage history
- **Advanced** — Self-hosted model benchmarks, notebook-style interface, multi-agent collaboration
- **Enterprise auth** — SAML and SCIM support
