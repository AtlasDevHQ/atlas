Integrate Atlas into a Nuxt 3 app using `@ai-sdk/vue`.

> **Prerequisites:** A running Atlas API server. See [Bring Your Own Frontend](/docs/frameworks/overview) for architecture and common setup.

<Callout type="warn" title="AI SDK v6 compatibility">
  The transport-based API (`DefaultChatTransport`, `sendMessage`) shown below is specific to `@ai-sdk/react` and may not be available in `@ai-sdk/vue`. If `useChat` from `@ai-sdk/vue` does not support the `transport` option in your version, use the [sync query endpoint](/docs/frameworks/overview#streaming-vs-synchronous) (`POST /api/v1/query`) instead -- it works with any HTTP client and returns a structured JSON response. The [`@useatlas/sdk`](https://www.npmjs.com/package/@useatlas/sdk) package provides a typed client for this endpoint.
</Callout>

---

## 1. Install dependencies

```bash
bun add @ai-sdk/vue ai
```

## 2. Configure the API URL

### Option A: Same-origin proxy (recommended)

Add a Nitro route rule in `nuxt.config.ts` to proxy `/api/**` to the Atlas API:

```typescript
// nuxt.config.ts
export default defineNuxtConfig({
  nitro: {
    routeRules: {
      "/api/**": {
        proxy: "http://localhost:3001/api/**",
      },
    },
  },
  runtimeConfig: {
    public: {
      atlasApiUrl: "", // empty = same-origin
    },
  },
});
```

### Option B: Cross-origin

```typescript
// nuxt.config.ts
export default defineNuxtConfig({
  runtimeConfig: {
    public: {
      atlasApiUrl: "http://localhost:3001",
    },
  },
});
```

```bash
# Atlas API .env
ATLAS_CORS_ORIGIN=http://localhost:3000
```

## 3. Chat composable

Create a composable that wraps `@ai-sdk/vue`'s `useChat`:

```typescript
// composables/useAtlasChat.ts
import { useChat } from "@ai-sdk/vue";
import { DefaultChatTransport } from "ai";

export function useAtlasChat() {
  const config = useRuntimeConfig();
  const apiUrl = config.public.atlasApiUrl || "";
  const apiKey = useState<string>("atlas-api-key", () => "");
  const input = ref("");

  const transport = computed(() => {
    const headers: Record<string, string> = {};
    if (apiKey.value) {
      headers["Authorization"] = `Bearer ${apiKey.value}`;
    }
    return new DefaultChatTransport({
      api: `${apiUrl}/api/chat`,
      headers,
    });
  });

  const { messages, sendMessage, status, error } = useChat({ transport });

  function handleSend() {
    if (!input.value.trim()) return;
    const text = input.value;
    input.value = "";
    sendMessage({ text });
  }

  return {
    messages,
    status,
    error,
    input,
    apiKey,
    apiUrl,
    handleSend,
  };
}
```

## 4. Rendering tool calls

See the [React/Vite guide](/docs/frameworks/react-vite#rendering-tool-calls) for a complete `ToolPart` implementation. The data shapes are identical -- adapt the JSX to Vue template syntax. The key shapes:

- **`executeSQL` output** -- `{ success, columns, rows, row_count, truncated }` on success, or `{ success: false, error }` on failure.
- **`explore` output** -- a plain string (stdout of the command).
- **`state`** -- check for `"output-available"` to know when the result is ready.

See the [Data Stream Protocol](/docs/frameworks/overview#data-stream-protocol) section for the full part structure and a concrete example.

## 5. Synchronous queries (alternative)

If you don't need streaming, use the JSON query endpoint instead:

```typescript
// composables/useAtlasQuery.ts
export async function queryAtlas(question: string) {
  const config = useRuntimeConfig();
  const apiUrl = config.public.atlasApiUrl || "";

  const res = await $fetch(`${apiUrl}/api/v1/query`, {
    method: "POST",
    body: { question },
    headers: {
      "Content-Type": "application/json",
    },
  });

  return res;
}
```

See [Bring Your Own Frontend](/docs/frameworks/overview) for the full architecture and what `@atlas/web` adds on top.
