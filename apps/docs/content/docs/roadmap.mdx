Atlas uses public semver starting at `0.0.x`. For real-time tracking, see the [GitHub Project Board](https://github.com/orgs/AtlasDevHQ/projects/2).

## Shipped

### Public Launch

Initial open-source release with admin user management, one-click Vercel deploy (Neon + AI Gateway), `@useatlas` packages on npm, and CI automation for template drift checks and starter repo sync.

### Adapter Plugin Refactor

Datasource adapters extracted from core into standalone plugins. The Plugin SDK gained `parserDialect` and `forbiddenPatterns` hooks, SQL validation and the connection registry became plugin-aware, and official plugins shipped for ClickHouse, Snowflake, DuckDB, and Salesforce.

### Python Data Science Sandbox

A sandboxed `executePython` tool for data analysis — runs Python with import guards, renders charts inline in the chat UI, and works across all sandbox backends (nsjail, sidecar, Vercel).

### 0.1.0 — Documentation & Developer Experience

Docs site launched at [docs.useatlas.dev](https://docs.useatlas.dev) with reference pages, integration guides, interactive API reference from OpenAPI, OG images, sitemap, and llms.txt. CLI gained `atlas doctor`, `atlas validate`, shell completions, and better first-run errors. Test coverage expanded across web UI, SDK integration, and E2E suites. Added CHANGELOG, CONTRIBUTING guide, issue/PR templates, and unified brand colors.

### 0.2.0 — Plugin Ecosystem

Official plugins published to npm under `@useatlas/*` (ClickHouse, Snowflake, DuckDB, Salesforce, Slack, MCP, and more). Plugin SDK gained testing utilities, a scaffold command (`bun create @useatlas/plugin`), multi-type plugin support, and schema migrations. Docs site added per-plugin reference pages, a filterable plugin directory, a composition guide, and a plugin cookbook.

### 0.3.0 — Admin Console & Operations

Self-service management and observability for the admin console. Connection CRUD with encrypted URLs at rest, action approval with bulk operations, user invite flow, plugin and settings configuration. Query analytics dashboard, health dashboard with per-component checks, token usage tracking per-user and per-conversation, and OpenTelemetry traces. Scheduled tasks gained create/edit forms, run history with filters, and delivery channel configuration.

## 0.4.0 — Chat Experience

The current milestone. Polish the end-user chat experience.

- **Chat polish** — Dark/light mode toggle with CSS variable theming, suggested follow-up questions, data export (CSV/Excel), mobile-responsive layout, saved queries
- **Discovery** — Visual schema explorer for analysts, expanded chart types (area, stacked bar, scatter)

---

## 0.5.0 — Governance & Sharing

Access control, audit improvements, and collaboration features for production teams.

- **Access control** — Advanced RLS policies (multi-column, array claims), session management, OIDC/OAuth provider docs
- **Audit** — Audit log export and search, data classification tags on query trails
- **Collaboration** — Conversation sharing with permission controls, semantic layer diff in admin console

---

## 0.6.0 — Embeddable & Integrations

Atlas as drop-in analytics infrastructure for other products.

- **Embedding** — Embeddable widget (`<script>` tag + React component), SDK streaming API
- **Integrations** — Microsoft Teams plugin, generic webhook plugin for Zapier/Make/n8n, email digest subscriptions
- **Data protection** — PII detection and column masking via plugin hooks

---

## 0.7.0 — Performance & Scale

Handle larger semantic layers, more concurrent users, and bigger result sets.

- **Caching** — Query result caching with configurable TTL, semantic layer indexing
- **Infrastructure** — Connection pool improvements, streaming Python output, tenant-scoped semantic layers
- **Datasources** — BigQuery plugin

---

## 0.8.0 — Intelligence & Learning

An agent that gets smarter over time.

- **Knowledge** — Curated prompt library per industry, query suggestion engine from usage history
- **Advanced** — Self-hosted model benchmarks, notebook-style interface, multi-agent collaboration
- **Enterprise auth** — SAML and SCIM support
