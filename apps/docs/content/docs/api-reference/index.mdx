Atlas exposes a JSON + streaming API over HTTP. All endpoints are served by the Hono API server (standalone on `:3001`, or proxied through Next.js on `:3000`).

## Base URL

| Setup | Base URL |
|-------|----------|
| Same-origin (Next.js rewrites) | `/api` |
| Cross-origin (standalone API) | `http://localhost:3001/api` |
| Production | `https://api.useatlas.dev/api` |

## Authentication

Authentication depends on the configured `ATLAS_AUTH_MODE`:

| Mode | Header |
|------|--------|
| `none` | No auth required |
| `api-key` | `Authorization: Bearer <ATLAS_API_KEY>` |
| `managed` | Session cookie (Better Auth) |
| `byot` | `Authorization: Bearer <JWT>` |

## Endpoints

### Core

- [**Chat**](/docs/api-reference/chat) — Stream agent responses via `POST /api/chat`
- [**Query**](/docs/api-reference/query) — Synchronous JSON query via `POST /api/v1/query`
- [**Conversations**](/docs/api-reference/conversations) — List, get, and delete conversations
- [**Health**](/docs/api-reference/health) — Health check and readiness probe

### Features

- [**Actions**](/docs/api-reference/actions) — Approval-gated write operations
- [**Scheduled Tasks**](/docs/api-reference/scheduled-tasks) — Recurring query schedules and delivery
- [**Auth**](/docs/api-reference/auth) — Better Auth endpoints (signup, signin, session)
- [**Slack**](/docs/api-reference/slack) — Slash commands, events, and OAuth
- [**Admin**](/docs/api-reference/admin) — Admin console API (users, settings, connections)
