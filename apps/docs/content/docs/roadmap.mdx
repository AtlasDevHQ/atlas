Atlas uses public semver starting at `0.0.x`. For real-time tracking, see [GitHub Issues](https://github.com/AtlasDevHQ/atlas/issues) and [Milestones](https://github.com/AtlasDevHQ/atlas/milestones).

## Shipped

### Public Launch

Initial open-source release with admin user management, one-click Vercel deploy (Neon + AI Gateway), `@useatlas` packages on npm, and CI automation for template drift checks and starter repo sync.

### Adapter Plugin Refactor

Datasource adapters extracted from core into standalone plugins. The Plugin SDK gained `parserDialect` and `forbiddenPatterns` hooks, SQL validation and the connection registry became plugin-aware, and official plugins shipped for ClickHouse, Snowflake, DuckDB, and Salesforce.

### Python Data Science Sandbox

A sandboxed `executePython` tool for data analysis — runs Python with import guards, renders charts inline in the chat UI, and works across all sandbox backends (nsjail, sidecar, Vercel).

### 0.1.0 — Documentation & Developer Experience

Docs site launched at [docs.useatlas.dev](https://docs.useatlas.dev) with reference pages, integration guides, interactive API reference from OpenAPI, OG images, sitemap, and llms.txt. CLI gained `atlas doctor`, `atlas validate`, shell completions, and better first-run errors. Test coverage expanded across web UI, SDK integration, and E2E suites. Added CHANGELOG, CONTRIBUTING guide, issue/PR templates, and unified brand colors.

### 0.2.0 — Plugin Ecosystem

Official plugins published to npm under `@useatlas/*` (ClickHouse, Snowflake, DuckDB, Salesforce, Slack, MCP, and more). Plugin SDK gained testing utilities, a scaffold command (`bun create @useatlas/plugin`), multi-type plugin support, and schema migrations. Docs site added per-plugin reference pages, a filterable plugin directory, a composition guide, and a plugin cookbook.

### 0.3.0 — Admin Console & Operations

Self-service management and observability for the admin console. Connection CRUD with encrypted URLs at rest, action approval with bulk operations, user invite flow, plugin and settings configuration. Query analytics dashboard, health dashboard with per-component checks, token usage tracking per-user and per-conversation, and OpenTelemetry traces. Scheduled tasks gained create/edit forms, run history with filters, and delivery channel configuration.

### 0.4.0 — Chat Experience

Dark/light mode toggle with persistence and admin-configurable brand color, suggested follow-up questions after each answer, Excel export alongside CSV, mobile-responsive chat UI, saved queries with starred filters, visual schema explorer for non-admin users, and expanded chart types (area, stacked bar, scatter).

### 0.5.0 — Launch

Embeddable widget infrastructure: `@useatlas/react` component with headless hooks API, `<script>` tag loader with programmatic JS API, and dedicated widget host route with configurable branding. SDK streaming via `streamQuery()` with abort support. BigQuery datasource plugin with cost-gated query approval. Conversation sharing with public links, OG meta tags, iframe embed mode, expiry controls, and org-scoped permissions. Launch prep including README overhaul with animated terminal recording, landing page refresh with interactive widget demo, and comparison pages. Extracted `@useatlas/types` shared package, hardened error handling across the stack, and isolated CLI test runner.

### 0.5.1 — Agent-Friendly Docs

Systematic docs overhaul for human and LLM consumption. Quick start rewritten with verification checkpoints. New how-to guides for widget customization, multi-datasource routing, schema evolution, and rate limiting. Reference pages gained inline-commented code examples, curl/fetch snippets, pagination and error handling patterns. Explanation pages added for SQL validation, semantic layer concepts, and integration comparisons. Agent optimization audit unified terminology, stabilized headings, and ensured no information exists only in images. Demo dataset comparison added to CLI and docs.

### 0.5.2 — Onboarding & CLI Polish

First 60 seconds made perfect. `atlas init` gained a progress spinner with ETA and a failure threshold for table profiling. First-run `.env` detection prompts users through setup. Pattern-matched errors replaced generic fallbacks for connection refused, provider unavailable, and query timeout scenarios. Internal errors now carry request IDs, and transient vs permanent error hints guide retry decisions. All CLI subcommands gained proper `--help` text and config validation messages became human-readable.

### 0.5.3 — UI & Accessibility Polish

ARIA labels on chat input, message bubbles, and skip links. Lighthouse/axe audit with automated accessibility checks added to CI. Full keyboard navigation for all interactive elements. Loading state audit ensuring every async operation has a visible indicator. Meaningful empty states, error boundaries with graceful fallback UI, and inline error feedback for conversation actions. Mobile polish pass at 375px–768px, chart responsiveness at all container sizes, and unified admin console table styles. Widget gained TypeScript type exports for embedders, CSS customization docs, and graceful error states for API down and auth failures.

### 0.5.4 — SDK & Integration Polish

SDK gained `listTables()` convenience method and a complete error code catalog with retry guidance. Streaming docs added for timeout/resume behavior and `retryAfterSeconds` null-safety. End-to-end integration tests added for the widget embed flow, SDK streaming, and MCP server. Every docs guide gained prerequisites and troubleshooting sections, reference pages were audited for code-first content, and a cross-link audit connected all related pages. Semantic discovery API now surfaces warnings when YAML parsing returns partial results. Obsidian plugin gained a README and docs page.

### 0.6.0 — Governance & Operational Hardening

Production-readiness hardening across infrastructure, governance, and integrations. Action timeout enforcement with `timed_out` status transitions, rollback API for reversible actions, configurable agent step limits and Python import guards, and operator-controlled explore backend priority. Advanced RLS with multi-column policies, array claims, and OR-logic between policies. Session management with admin listing/revoking and idle/absolute timeouts. Audit log gained CSV export, full-text search, connection/table filtering, and data classification tags extracted from the SQL AST. Plugin SDK shipped typed peer dep re-exports, beforeToolCall/afterToolCall hooks, and custom query validation for non-SQL datasources. SDK added `validateSQL()` and `getAuditLog()` methods. New interaction plugins for Microsoft Teams (Bot Framework, Adaptive Cards), generic webhooks (Zapier/Make/n8n), and email digest subscriptions. Admin console gained a semantic layer diff view comparing DB schema against YAML entities.

### 0.7.0 — Performance & Multi-Tenancy

Multi-tenant foundation via Better Auth organization plugin with org-scoped semantic layers, connections, caching, and explore tool. Tenant-scoped connection pooling with health-based drain and per-org metrics. Query result caching with configurable TTL and admin flush UI. Pre-computed semantic index for faster agent context loading with `atlas index` CLI. Streaming Python execution output with progressive stdout and inline chart renders. `atlas learn` CLI for offline semantic layer improvements — analyzes audit logs and proposes YAML amendments for query patterns, join discoveries, and glossary terms.

### 0.7.1 — Immediate Cleanup

Post-sprint quality pass on 0.7.0 code. Resolved lint warnings (unused exports, type params, variables), moved chat error stack traces to debug-level logging, filled docs gaps for `atlas index` CLI, streaming Python, and cache admin UI. Code review of new modules surfaced two refactoring issues tracked for 0.7.2.

### 0.7.2 — Type Safety & Code Smells

Systematic code quality pass across all packages. Eliminated non-null assertion operators, replaced explicit `any` types with proper typing, removed dead exports and unused code (net -1150 lines), and broke up complex functions over 50 lines into well-named helpers. Extracted shared Python wrapper code and fixed input mutation in the CLI learn module.

### 0.7.3 — Error Handling & Resilience

Comprehensive error handling audit across all packages. Eliminated silent catch blocks and standardized error type narrowing with `instanceof Error` guards. Replaced generic "Something went wrong" messages with actionable, context-specific guidance and added request IDs to all 500 responses. Audited fallback patterns for silent degradation and added logging where errors were being swallowed. Extended React error boundary coverage to organization context, streaming Python results, and shared conversations.

### 0.7.4 — Test Hardening

Test coverage expansion for 0.7.0 features. Added password endpoint tests (status check, change workflow, validation, DB error paths). Cache edge case tests for TTL boundaries, concurrent access, oversized entries, and LRU eviction accuracy. Streaming Python timeout and SSE protocol error path tests. `atlas learn` edge case tests for malformed audit entries, conflicting proposals, and full pipeline integration. Migrated remaining inline connection mocks to the shared `createConnectionMock` factory for consistency.

### 0.7.5 — Docs Completeness

Final 0.7.x release closing the refinement arc. Comprehensive feature-to-docs mapping audit across milestones 0.1.0–0.7.4, filling coverage gaps with new guide pages for query caching and multi-tenancy setup. Stale reference cleanup verified all API endpoints, config options, and environment variables against source code. Landing page refreshed with multi-tenancy, caching, and learning capabilities. Python tool guide expanded with streaming output and sandbox configuration.

### 0.8.0 — Intelligence & Learning

Dynamic learning layer that gets smarter with every query. Agent proposes learned patterns after successful queries with deduplication and confidence scoring; approved patterns are injected into agent context with relevance filtering. Admin review/approve/reject UI with bulk operations. Curated per-industry prompt library (SaaS, e-commerce, cybersecurity) with admin CRUD and chat browser. Query suggestion engine mining the audit log for ranked suggestions with contextual display. Self-hosted model improvements with test matrix for Ollama/vLLM/TGI. Notebook-style interface for cell-based exploratory analysis with re-run, keyboard navigation, and persistent state.

### 0.8.1 — Notebook Refinement

Comprehensive polish of the notebook UI. Error boundaries and shared transport hook for stability. Error propagation refactored so callers can distinguish failure reasons. Keyboard navigation tests, UX polish (keybinding consistency, auto-expand, dialog dedup), and hook-level test coverage. Missing error codes added to type catalog, reference gaps filled for learn config and CLI flags, and OpenAPI spec updated with 50+ missing endpoints via codegen pipeline. Notebook Phase 2 shipped fork/branch from any cell, drag-and-drop reorder with server persistence, and a branch selector. Phase 3 added markdown text cells for narrative annotation and export to Markdown/HTML.
