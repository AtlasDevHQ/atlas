For a `multi`-scope client, removes a single (clientId, workspaceId) grant. The OAuth client itself stays intact for the other granted workspaces; tokens that have already been minted stay valid for those. Cross-user isolation: the caller must own the OAuth client (registered by them) AND be a current member of the workspace whose grant they're removing.
